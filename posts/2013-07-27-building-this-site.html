Building this Site

On July 27, 2013

I thought I’d give a run down of how this site was built, and what it runs on.

Hakyll

Hakyll is a Haskell based static site generator. I picked Hakyll because at the time I decided to make a website, I was learning Haskell and it seemed like a fun project. There are number of good tutorials on Hakyll, but I found the most useful thing was to check out read other’s sources.

Why Static?

I decided to make a static website for a few reasons. The biggest thing for me was ease of hosting. As there are no real security issues or maintenance issues, it simplifies running the site to just making and publishing the content. At the same time, I retain control over that content.

Hosting

The website is hosted on Github pages, which provides free hosting for static web pages. So far it’s worked pretty well. It provides basically everything you need for a static site such as a cname, favicon, or a custom 404 page. I haven’t noticed any major disruptions, although as it’s a personal website it’s not exactly in high demand.

One of the nice parts of working with Github pages is the simple work-flow. In order to publish a new page, I just have to commit and push it to the site’s Github repo.

Design

The basic design owes a lot to learn you a haskell, an excellent tutorial on Haskell. As it says in the footer, the background is by Sander Ottens. I found that on subtle patterns, which is a good source for minimalist backgrounds.

Comments and Analytics

As it’s a static website, the only way to add more advanced such as comments or analytics is to add some external javascript. At the beginning of the year I added disqus comments, and google analytics.

The biggest problem for me with these was that they add a bunch of not particularly light javascript. Disqus was the offender in this regard. Given one of the benefits of having a static site in the first place is a fast page-load, I decided to just cut it.
